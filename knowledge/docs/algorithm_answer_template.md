# 算法题讲解模板

当你给别人讲算法题时，可以尽量用一个固定模板，回答会更清晰。

## 推荐模板

1. 题意理解
2. 核心思路
3. 状态定义或数据结构
4. 转移 / 计算过程
5. 复杂度分析
6. 代码实现

## 对新手更友好的说法

- 先说“为什么要这么做”
- 再说“怎么做”
- 最后说“代码怎么写”

## 适合 AlgoGuide Agent 的回答风格

这个项目的回答风格应该尽量接近：

- 先解释直觉
- 再给步骤
- 必要时补简短代码
- 如果用户说“讲简单一点”，就减少术语

## 示例开头

你可以这样开头：

“这题的关键是把问题转成一个更容易处理的形式，然后再用动态规划 / 前缀和 / BFS 去解决。”
